# 智慧交易运营管理平台 — 系统全景总结

## 一、系统定位

面向电力交易市场的**多省网一体化运营管控平台**，覆盖电力现货交易全流程：电价预测 → 报价策略 → 交易执行 → 结算分析 → 成本管控。已部署9个省网（山西、河南、湖北、蒙西、蒙东、宁夏、甘肃、陕西、云南、广西）。

**访问地址**: http://10.60.64.219:20690/qctc/

## 六、算法与模型

### 6.1 算法架构

```
Java算法调度服务 (bdss-algorithm-biz.jar)
  │
  ├── AlgorithmFactory ──→ 根据配置选择算法实现
  │     ├── XGBoost V1/V2/V3
  │     ├── LightGBM
  │     ├── Cluster+XGBoost
  │     ├── PSO粒子群
  │     └── BestParams寻优
  │
  └── ProcessBuilder ──→ 调用Python脚本
        ├── Python 3.6 (河南专用)
        └── Python 3.9 (通用新版)
```

**交互机制**: Java组装参数 → 写CSV到共享目录 → 调起Python命令行 → Python读CSV → 模型推理 → 写CSV结果 → Java解析CSV → 存MySQL

## 八、请求链路

以"日前电价预测"为例：

```
1. 用户在前端选择"日前电价预测"
2. Vue → Axios → POST /qctc/algo/price/forecast
3. Nginx(:20690) → proxy_pass → Gateway(:40001)
4. Gateway → 鉴权 → 路由到 bdss-algorithm-biz(:9099)
5. AlgoPriceForecastController 接收请求
6. AlgorithmFactory 根据租户配置选择算法实现（如XGBoost）
7. AlgorithmXGBoostCnServiceImpl:
   a. 从MySQL读取模型参数(algo_param_xgboost)
   b. 从MySQL读取特征配置(algo_param_feature)
   c. 从MySQL读取执行路径(algo_param_url)
   d. 组装Python命令 + 参数
   e. ProcessBuilder → python3 /opt/sxpython/.../run.pyc -pr sx -m ...
8. Python脚本:
   a. 读取CSV输入(边界/历史电价/负荷/气象)
   b. XGBoost模型加载/推理
   c. 输出预测结果CSV
9. Java解析CSV → 存入algo_price_forecast_clearing表
10. 返回预测结果 → 前端ECharts渲染电价曲线
```

## 十、系统特点总结

1. **多省网一体化**: 一套系统覆盖9个省网，通过tenant_id隔离
2. **Java-Python混合架构**: Java调度+Python推理，灵活适配不同算法
3. **算法可配置**: 所有模型参数、特征、路径通过数据库配置，无需改代码
4. **算法热更新**: Python脚本(.pyc)通过管理界面上传，支持在线更新
5. **双层Python环境**: 3.6适配旧模型（河南），3.9支持新特性（强化学习等）
6. **全流程覆盖**: 预测→报价→交易→结算→成本，电力交易闭环
7. **高并发就绪**: 3000 MySQL连接、Redis缓存、Nacos动态配置
