# P0 阶段任务拆解 — 指标中心核心功能

> P0 目标：完成50个核心指标的完整链路，覆盖电量/电价/电费三大类，跑通「原始数据抽取 → 公式计算 → 日聚合 → 前端展示」全流程。

## P0 指标范围

| 指标组 | 数量 | 覆盖指标 |
|--------|------|---------|
| **核心电量** | 18 | 合约出清/日前出清/实时出清/上网/偏差(正负)/省间 |
| **核心电价** | 12 | 合约/日前/实时/偏差(正负)/现货均价/总平均 |
| **核心电费** | 20 | 合约/偏差(正负)/全量/差价合约/总电费 |

计算层级覆盖 L0(原始) → L1(基础) → L2(偏差) → L3(电费)，共4层。

## Sprint 3: 数据抽取与公式引擎 (Week 3)

### T3.1 动态数据抽取服务
- [ ] `DataExtractService` — 读取 `ind_data_source` 配置，动态构建SQL
- [ ] `@DS` 动态数据源切换 — 从11个业务库读取原始数据
- [ ] 实现P0需要的28个原始指标数据抽取:
  - **现货市场库**: `spot_ahead_trade`(日前出清), `spot_real_trade`(实时出清)
  - **交易管理库**: `trxn_contract_match_month`(合约电量/电价)
  - **结算管理库**: `sett_unit_bill_hour`(结算电量/电价/电费), `sett_power_hour_unit`(上网电量)
  - **基础参数库**: `bas_unit`(额定容量/厂用电率)
  - **算法管理库**: `algo_price_forecast_clearing`(预测电价)
- [ ] 抽取结果写入 `ind_data_timeseries`
- **验收**: 调用抽取服务后，`ind_data_timeseries` 有96点分时数据

### T3.2 公式引擎集成

- [ ] 异常处理：单指标失败不阻塞其他指标
