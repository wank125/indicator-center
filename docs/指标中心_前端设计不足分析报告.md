# 指标管理平台前端设计不足分析报告

> 基于对31个 Vue 组件、Pinia Store、API层、Mock数据层的逐行代码阅读，以下是基于实际代码的精准缺陷分析，非泛泛而谈。

---

## 一、功能流程断裂（最根本缺陷）

### 🔴 问题1：指标树与查询联动逻辑错误

**代码位置**：`query/Index.vue:102`

```javascript
// 当前代码：
const matched = MOCK_INDICATOR_OPTIONS.find((o) => o.code === data.dimCode)
if (matched) {
  store.setSelectedCode(matched.code)
  handleQuery()
}
```

**问题**：维度树节点的 `dimCode` 是分类编码（如 `CLEARING_ENERGY`），而非指标编码（如 `DAM_CLEARING_ENERGY`）。当前用 `dimCode === 指标code` 来匹配，**只有精确碰巧相同时才能触发查询**，大部分叶子节点点击后没有任何响应。

**正确做法**：树叶节点应携带 `indicatorCode` 字段，或树数据结构本身区分「分类节点」和「指标节点」。

---

## 二、数据展示能力不足

### 🟡 问题6：DataTable 过度简化（3列）

**代码位置**：`query/DataTable.vue`（21行，全文）

```html
<el-table-column prop="date" label="日期" width="140" />
<el-table-column prop="timePoint" label="时间点" width="100">
<el-table-column prop="value" label="数值" align="right">
```

**问题**：
- 只有3列：日期/时间点/数值，没有单位、数值格式化、环比列
- 没有合计行（电费总额、电量汇总等关键合计）
- 数值显示为原始 number，没有按指标类型格式化（电价2位小数、电费千分位等）
- 无排序功能

---

### 🟡 问题7：IndicatorChart 图表能力严重不足

**代码位置**：`query/IndicatorChart.vue:33-60`

```javascript
// 只有单一系列，没有 legend
chart.setOption({
  tooltip: { trigger: 'axis' },
  grid: { left: 60, right: 20, top: 20, bottom: 40 },
  xAxis: { type: 'category', data: props.xData },
  yAxis: { type: 'value', scale: true },
  series: [seriesBase],  // 只有一条线，没有legend
}, true)
```

**问题**：
- **无 dataZoom**：96点分时数据横向密集，无法缩放查看局部细节
- **无 legend**：多指标时看不出哪条线是哪个指标
- **无 markLine/markPoint**：无法标注峰谷时段、异常点
- **无 tooltip formatter**：数值显示不带单位，无格式化
- **图表高度固定320px**：不可拖拽调整，长周期数据展示效果差
- **y轴无单位标注**：元/MWh、MWh、元等无法区分

---

### 🟡 问题8：CompareTable 缺少关键列

**代码位置**：`query/CompareTable.vue`

当前列：`期间 | 当前值 | 对比值 | 变化率`

**缺失**：
- 没有**绝对差值**列（当前值 - 对比值 = 差多少）
- 没有**趋势方向图标**（↑↓ 箭头配色）
- 没有**分析维度**（只能按全部，无法按机组/电厂维度对比）
- 变化率颜色逻辑错误：电力行业里负偏差增大（数值变负更大）不一定是"坏"，当前简单用正绿负红会误导分析师

---

### 🟡 问题9：KpiCard 无加载骨架屏

**代码位置**：`dashboard/KpiCard.vue`

```html
<!-- 当 value 为 null 时只显示 '--'，没有骨架屏 -->
<span class="kpi-value">{{ formattedValue }}</span>
```

**问题**：
- 初始加载时 KPI 卡片突然从 '--' 跳变为数字，体验生硬
- 没有 `v-loading` 骨架屏状态
- 没有数据异常状态（如计算失败时应显示"数据异常"而非'--'）
- KPI卡片不支持点击下钻（点击"总边际贡献"应跳转至对应指标查询页）

## 四、组件设计质量问题

### 🟡 问题14：FormulaEditor 依赖图是静态假图

**代码位置**：`config/DependencyGraph.vue`（2014字节，极简）

```javascript
// 依赖拓扑图是用 SVG 画的示意图，不是真实依赖关系
// 节点位置硬编码，不会根据实际公式依赖动态生成
```

**问题**：依赖拓扑图是一个静态的 SVG 示意图，无论选什么指标，图形都一样。真实的依赖关系需要从后端获取并用 D3.js 或 ECharts Graph 动态渲染。

---

### 🟡 问题15：公式编辑器没有语法高亮

**代码位置**：`config/FormulaEditor.vue:22-24`

```html
<el-input v-model="formula" type="textarea" :rows="3"
  placeholder="输入公式表达式，如: {DAM_CLEARING_ENERGY} - {MLT_CONTRACT_ENERGY}" />
```

**问题**：设计规格书要求「语法高亮：指标引用(青色)/运算符(黄色)/函数(白色)」，实际实现只是一个普通 textarea，没有任何高亮。添加指标依赖时也不自动插入 `{CODE}`，而是重新生成整个公式。

---

### 🟡 问题16：SourceMapping 保存不持久化

**代码位置**：`config/SourceMapping.vue`（对应实施方案记录）

```javascript
function handleSave() {
  ElMessage.success('数据源配置已保存')  // 同样只弹提示，无 API 调用
}
```

**问题**：数据源映射是指标中心的核心配置，但保存操作不调用后端接口，重新打开页面配置丢失。

## 六、工程化规范缺陷

### 🟡 问题19：Mock 降级策略污染了正常错误处理

**代码位置**：`stores/indicator.ts:64-78`

```javascript
async function loadTimeseries(code: string, startDate: string, endDate: string) {
  ...
  try {
    timeseriesData.value = await getTimeseries(...)
  } catch {   // ← catch 没有区分错误类型
    timeseriesData.value = generateMockTimeseries()  // 任何错误都用 Mock 覆盖
  }
}
```

**问题**：网络错误、401未授权、500服务异常、数据格式错误，全部被 `catch` 静默降级到 Mock 数据。用户看到数据展示正常，实际上后端可能已经崩溃。正确做法：只在开发模式或特定降级标志下使用 Mock。

---

### 🟡 问题20：指标选择器使用 Mock 数据

**代码位置**：`components/IndicatorSelect.vue`（从 mock-data 导入选项列表）

```javascript
import { MOCK_INDICATOR_OPTIONS } from '@/api/mock-data'
```

**问题**：工具栏的指标搜索下拉框，数据来自本地的静态 Mock 列表，没有接入 `/qctc/indicator/definitions` 接口，导致：
1. 搜索结果固定，不反映真实的199个指标
2. 新增或修改指标后，下拉框不会更新
