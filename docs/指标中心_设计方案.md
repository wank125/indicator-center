# 指标中心设计方案

## 一、现状分析

### 1.1 指标资产盘点

| 维度 | 数量 | 说明 |
|------|------|------|
| 系统已入库指标 | 116个 | 6大类：市场边界/电价/交易/现货/结算/成本，已映射到30+张数据库表 |
| 客户需求指标库 | 199个 | 发电侧+集团公司，覆盖中长期/现货/省间/辅助服务/容量/绿色权益6个市场 |
| 系统精确匹配 | 8个 (4%) | 直接可用 |
| 系统有原始数据 | 38个 (19%) | 数据已入库，需配置计算逻辑 |
| 可由已有指标计算 | 26个 (13%) | 基于已有指标做加权/拆分/同比环比 |
| 可由算法产出 | 8个 (4%) | 算法预测/结算服务可产出 |
| 需二次开发 | 118个 (59%) | 需编写计算逻辑或接入新数据源 |

### 1.2 核心问题

1. **指标散落各处**：199个指标分布在11个微服务、30+数据库、100+张表中，无统一管理入口
2. **计算逻辑硬编码**：偏差电费、边际贡献等118个指标的计算逻辑散落在各服务代码中
3. **缺少统一语义层**：同一指标在不同模块有不同命名（如"日前出清电量"在现货模块和结算模块字段不同）
4. **无法灵活扩展**：新增指标需改代码、改数据库、改前端，周期长
5. **同环比/排名无法通用化**：26个同环比指标和5个TOP10排名逻辑重复开发

---

## 二、指标中心定位与目标

### 2.1 定位

**指标中心是电力交易系统的"指标中枢"**，统一管理指标定义、计算逻辑、数据存储和查询服务，为前端展示、报表分析和决策支持提供标准化、可配置的指标数据服务。

### 2.2 目标

| 目标 | 描述 |
|------|------|
| 统一管理 | 199+指标统一定义、统一编码、统一口径 |
| 配置化计算 | 计算指标通过公式配置而非硬编码，新增指标零代码 |
| 实时可用 | 96点分时数据T+1产出，日/月聚合实时查询 |
| 多维分析 | 支持按省网/机组/时间/市场类型多维度下钻 |
| 扩展灵活 | 支持未来新省网、新市场类型指标快速接入 |

### 3.2 与现有系统集成方式

指标中心作为**独立微服务**接入现有Spring Cloud体系，不侵入现有业务服务：

```
现有服务(数据生产者)                    指标中心(数据消费者)
─────────────────────                ──────────────────
bdss-algorithm-biz  ─── 算法预测 ──→  读取 algo_price_forecast_clearing
bdss-spot-market    ─── 现货出清 ──→  读取 spot_ahead_trade / spot_real_trade
bdss-transaction    ─── 中长期   ──→  读取 trxn_contract_match_month
bdss-settlement     ─── 结算数据 ──→  读取 sett_unit_bill_hour / sett_base_subject
bdss-cost           ─── 成本数据 ──→  读取 cost_day_indicator / cost_sis_value
bdss-market-analysis── 市场边界  ──→  读取 mkt_netload / mkt_unifyprice_*
bdss-group-data     ─── 集团数据 ──→ 读取 group_sett_unit_bill_day
bdss-basic-parameter── 基础参数  ──→  读取 bas_unit (机组信息)
```

**集成原则**：
- 只读取现有数据库，不修改现有表结构
- 通过数据库直连或Feign调用获取原始数据
- 计算结果写入指标中心自有库

## 五、计算引擎设计

### 5.1 计算类型分类

基于199个指标的计算逻辑，抽象出5种计算类型：

| 类型 | 编号 | 说明 | 示例指标数量 |
|------|------|------|------------|
| **四则运算** | CALC_ARITH | A * B, A + B - C 等 | ~50个 |
| **条件分支** | CALC_COND | IF(A>=0, A, 0) 正负偏差拆分 | ~25个 |
| **聚合汇总** | CALC_AGG | SUM/AVG 按日/月/年聚合 | ~35个 |
| **同环比** | CALC_YOY_MOM | (本期-同期)/同期 | ~24个 |
| **排名** | CALC_RANK | 按维度排序取TOP N | ~5个 |

### 5.2 公式表达式设计

采用**占位符+运算符**的方式定义公式，指标中心通过指标编码引用：

```
# 四则运算示例
日前全电量电费 = {DAM_CLEARING_ENERGY} * {DAM_CLEARING_PRICE}
合约电费 = {BASE_FEE} + {MLT_FEE}
现货偏差电量 = {DAM_DEVIATION} + {RTM_DEVIATION}

# 条件分支示例
日前正偏差电量 = IF({DAM_DEVIATION} >= 0, {DAM_DEVIATION}, 0)
现货正偏差电费 = IF({SPOT_DEVIATION} >= 0, {SPOT_DEVIATION_FEE}, 0)

# 聚合示例
日结算电量 = SUM({SETTLE_ENERGY_HOURLY}) GROUP BY unit_id, trade_date
月度电价 = SUM({MONTHLY_FEE}) / SUM({MONTHLY_ENERGY})

# 同环比示例
电量同比 = ({ENERGY} - {ENERGY:YOY}) / {ENERGY:YOY}

# 排名示例
收益TOP10 = RANK({TOTAL_FEE}, 10, DESC) GROUP BY group_company, plant
```

### 5.3 计算调度流程

```
每日 T+1 02:00 自动触发
         │
         ▼
┌─────────────────┐
│ 1. 依赖图构建    │  从 ind_calc_formula 读取所有指标公式

└────────┬────────┘

└────────┬────────┘
```

### 5.4 计算优先级（拓扑排序层级）

```
第0层 (原始数据)：
  基数出清电量, 中长期出清电量, 日前出清电量, 实时出清电量, 上网电量,
  基数出清电价, 中长期出清电价, 日前出清电价, 实时出清电价,
  额定容量, 厂用电率, 最小技术出力, 实时单位变动成本,
  计划收益, 计划发电量, 装机容量, 有效容量 ...

第1层 (基础计算)：
  合约出清电量 = 基数出清 + 中长期出清
  基数出力 = 基数出清电量 / (1 - 厂用电率) / 4
  中长期出力 = 中长期出清电量 / (1 - 厂用电率) / 4
  日前出清电力 = 日前出清电量 / (1 - 厂用电率) / 4
  实时出清电力 = 实时出清电量 / (1 - 厂用电率) / 4

第2层 (偏差计算)：
  日前偏差电量 = 日前出清 - 合约出清
  实时偏差电量 = 上网电量 - 日前出清
  现货偏差电量 = 日前偏差 + 实时偏差
  日前正/负偏差电量 = IF(日前偏差 >= 0, 日前偏差, 0) / IF(日前偏差 < 0, 日前偏差, 0)
  ...

第3层 (电费计算)：
  基数电费 = 基数出清电价 × 基数出清电量
  合约电费 = 基数电费 + 中长期电费
  日前全电量电费 = 日前出清电量 × 日前出清电价
  日前偏差电费 = 日前偏差电量 × 日前出清电价
  现货全电量电费 = 日前全量电费 + 实时偏差电量 × 实时出清电价
  总电费 = 合约电费 + 日前偏差电费 + 实时偏差电费
  ...

第4层 (差价合约)：
  基数差价合约电费 = 基数出清电量 × (基数出清电价 - 日前出清电价)
  合约差价合约电费 = 基数差价 + 中长期差价
  日前差价合约电费 = 日前偏差电量 × (日前出清电价 - 实时出清电价)
  ...

第5层 (边际贡献)：
  合约变动成本额 = 实时单位变动成本 × 合约电量
  合约边际贡献 = 合约电费 - 合约变动成本额
  日前偏差边际贡献 = 日前偏差电费 - 日前偏差电量 × 实时单位变动成本
  总边际贡献 = 总电费 - 实时变动成本额
  ...

第6层 (负荷率/覆盖率)：
  基数负荷率 = 基数出清电量 / (额定容量 × (1-厂用电率) × 4) × 100%
  合约覆盖率 = 合约电量 / 上网电量 × 100%
  ...

第7层 (综合指标)：
  度电收益 = 总电费 / 上网电量
  可利用小时数 = 上网电量 / 装机容量
  收入完成率 = 实际收入 / 计划收入 × 100%
  现货均价 = 现货全电量电费 / 上网电量
  ...

第8层 (同环比)：
  电量同比 = (本期电量 - 去年同期电量) / 去年同期电量 × 100%
  电费环比 = (本期电费 - 上期电费) / 上期电费 × 100%
  ...

第9层 (排名)：
  收益TOP10, 度电收益TOP10, 发电量TOP10, ...
```

## 六、指标编码规范

### 6.1 编码规则

```
{市场}_{分类}_{指标}_{正负}_{日/月/年}

市场前缀：
  MLT  = 中长期市场 (Medium-Long Term)
  DAM  = 日前市场 (Day-Ahead Market)
  RTM  = 实时市场 (Real-Time Market)
  INT  = 省间 (Inter-provincial)
  AUX  = 辅助服务 (Auxiliary Service)
  CAP  = 容量市场 (Capacity)
  GRN  = 绿色权益 (Green)
  GRP  = 集团聚合 (Group)

分类前缀：
  PWR  = 电量 (Power/Energy)
  PRC  = 电价 (Price)
  FEE  = 电费 (Fee)
  DEV  = 偏差 (Deviation)
  CNT  = 贡献 (Contribution)
  COST = 成本 (Cost)
  RATE = 负荷率/覆盖率 (Rate)
  HRS  = 可利用小时 (Hours)
```

### 6.2 编码示例

| 编码 | 指标名称 | 公式 |
|------|---------|------|
| `MLT_BASE_ENERGY` | 基数出清电量 | 原始数据 |
| `MLT_MLT_ENERGY` | 中长期出清电量 | 原始数据 |
| `MLT_CONTRACT_ENERGY` | 合约出清电量 | `{MLT_BASE_ENERGY} + {MLT_MLT_ENERGY}` |
| `DAM_CLEARING_ENERGY` | 日前出清电量 | 原始数据 |
| `RTM_CLEARING_ENERGY` | 实时出清电量 | 原始数据 |
| `DAM_DEV_ENERGY` | 日前偏差电量 | `{DAM_CLEARING_ENERGY} - {MLT_CONTRACT_ENERGY}` |
| `RTM_DEV_ENERGY` | 实时偏差电量 | `{ONLINE_ENERGY} - {DAM_CLEARING_ENERGY}` |
| `SPOT_DEV_ENERGY` | 现货偏差电量 | `{DAM_DEV_ENERGY} + {RTM_DEV_ENERGY}` |
| `DAM_DEV_POS_ENERGY` | 日前正偏差电量 | `IF({DAM_DEV_ENERGY}>=0, {DAM_DEV_ENERGY}, 0)` |
| `DAM_DEV_NEG_ENERGY` | 日前负偏差电量 | `IF({DAM_DEV_ENERGY}<0, {DAM_DEV_ENERGY}, 0)` |
| `MLT_BASE_FEE` | 基数电费 | `{MLT_BASE_PRICE} * {MLT_BASE_ENERGY}` |
| `MLT_CONTRACT_FEE` | 合约电费 | `{MLT_BASE_FEE} + {MLT_MLT_FEE}` |
| `DAM_FULL_FEE` | 日前全电量电费 | `{DAM_CLEARING_ENERGY} * {DAM_CLEARING_PRICE}` |
| `TOTAL_FEE` | 总电费 | `{MLT_CONTRACT_FEE} + {DAM_DEV_FEE} + {RTM_DEV_FEE}` |
| `SPOT_AVG_PRICE` | 现货均价 | `{SPOT_FULL_FEE} / {ONLINE_ENERGY}` |
| `MLT_BASE_RATE` | 基数负荷率 | `{MLT_BASE_ENERGY} / ({RATED_CAP}*(1-{AUX_RATE})*4) * 100` |
| `COST_MARGIN` | 总边际贡献 | `{TOTAL_FEE} - {RTM_VAR_COST}` |
| `PER_UNIT_REVENUE` | 度电收益 | `{TOTAL_FEE} / {ONLINE_ENERGY}` |
